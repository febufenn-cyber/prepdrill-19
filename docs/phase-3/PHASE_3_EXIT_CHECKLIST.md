# Phase 3 Exit Checklist

## Software

- [x] Mandatory fresh-source manifest roles are enforced.
- [x] Manifest fingerprints are deterministic.
- [x] Count mismatches and orphaned assets/contexts block activation.
- [x] Fixture, legacy, generated, repaired-unreviewed, invalid-source, unresolved-rights, and blocking-duplicate counts block activation.
- [x] Golden-set and audit minimums are enforced.
- [x] Phase 1.5 gate and corpus fingerprints must be current.
- [x] Review-cost evidence is required.
- [x] Authorization requires a named owner and reason.
- [x] Corpus drift invalidates authorization.
- [x] Migration rollback is append-only.
- [x] Supabase evidence tables expose no learner policy.
- [x] Phase 3 evaluator and inherited workflows pass.

## Real evidence and activation

- [ ] Fresh UGC NET Paper 1 source delivery registered.
- [ ] Actual record/context/asset/key/taxonomy/source counts reconciled.
- [ ] Real 100-question golden set verified.
- [ ] Real 250-question audit completed.
- [ ] Rights, answer, rendering, provenance, mapping, and duplicate evidence resolved.
- [ ] Review cost measured.
- [ ] Current Phase 1.5 gate passed.
- [ ] Named release owner authorized the current fingerprint.

Phase 3 software completion does not claim production activation.
