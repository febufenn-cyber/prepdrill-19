# Phase 14 Blind Spots

- A shared codebase does not guarantee subject isolation; every ID, cache key, gate, attempt, state, and export must carry a subject namespace.
- Paper 1 authorization cannot be reused as evidence for Paper 2 English.
- Client parity should compare server-derived event/state fingerprints, not screenshots alone.
- Platform-specific offline caches can resurrect stale answers or sessions unless the server contract includes revision and state versions.
- Onboarding names are profile data, not identity; authenticated account ownership remains authoritative on all clients.
- A renderer must declare unsupported block types before session creation rather than failing mid-question.
- Configuration enables software reuse but does not authorize a new exam launch.
- Paper 1 trust, retention, economics, and operations can regress while expansion metrics look attractive.
- A deterministic scale decision still requires real measured inputs; synthetic values prove only the scorecard logic.
