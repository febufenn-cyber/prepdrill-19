# Phase 0 Prototype Contract

Create a disposable, clickable prototype—not production UI—with six screens:

1. Promise/landing
2. UGC NET Paper 1 confirmation
3. Five-question player
4. Immediate wrong-answer correction
5. Result showing evidence confidence
6. Recommended next drill with reason

## Required states

- correct answer;
- incorrect answer;
- insufficient evidence;
- likely weakness with evidence;
- content-report action;
- guest progress save prompt after value.

## Test constraints

- First answer reachable in under 60 seconds.
- No mandatory registration before answering.
- Avoid a percentage mastery score from sparse evidence.
- Recommendation explanation must be visible.
- Use realistic difficult question formats, not only simple text MCQs.

Research artefacts should record participant ID, task completion, confusion, interpretation of the recommendation and targeted-vs-random preference.
